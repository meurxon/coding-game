Command Chaining with ;

Use a semicolon ; to run multiple commands in a single line. Each command executes one after another, left to right. If one command fails or does nothing, the rest still run. There is no limit on how many commands you can chain.

How it works internally: When you type a line with ;, the terminal splits it at each semicolon and runs each piece as its own separate command. The commands don't know about each other - they just run in order. State changes from earlier commands (like spending energy) ARE visible to later commands.

# Chain multiple game actions together
run mine() ; run mine() ; run mine() ; run compile()

# Use conditions with loops
if energy > 20 run hack() ; if data >= 50 run compile()

# Full auto-cycle: mine, compile, sleep when empty
run mine() ; run compile() ; if energy = 0 run sleep()

# Check status after actions
run mine() ; run hack() ; run scan() ; status()

Important rules about chaining:

• If one command in a chain fails (e.g. not enough energy for run mine()), the rest still execute.

• You can chain: run, if, loop, function, status, help, sleep, print, etc.

• The if command only controls the single next command after it. Commands after the ; run regardless of the condition.

• State changes carry through: if you run mine() first, the data gained is visible to a later run compile().

• You can chain as many commands as you want. There is no limit.

• Spaces around ; are optional. cmd1;cmd2 works the same as cmd1 ; cmd2.

if (Conditional Commands)

The if command checks a condition and only runs the next command if the condition is true. If false, it skips that command and prints "FALSE (skipped)". The command after the condition must be a single command (but that command can itself be a chain using ;).

How if controls commands - CRITICAL to understand:

The if command controls the single next command after it. When you chain commands with ;, only the command immediately after if is conditional. Everything after the next ; runs regardless.

# WRONG - only "run sleep()" is conditional, "run mine()" ALWAYS runs
if energy = 0 run sleep() ; run mine()

# RIGHT - both commands are conditional with separate ifs
if energy = 0 run sleep() ; if energy = 0 run mine()

# RIGHT - combine them into one chained command after if
if energy = 0 run sleep() ; run mine()

# The above works because "run sleep() ; run mine()" is parsed as:
# if energy = 0  ->  run sleep()
# ; run mine()  ->  always runs (outside the if)

# To make BOTH conditional, use two separate ifs:
if energy = 0 run sleep() ; if energy = 0 run mine()

Practical automation patterns:

Important notes:

• = and == both mean "equal to". Use = for simplicity.

• != means "not equal to". Example: if energy != 0 run mine() only mines if you have energy.

• The value must be a number. You cannot compare to text or other variables.

• If the stat name is wrong, you'll get an error listing the valid stats.

• The condition is checked at the moment the if command runs. If a previous command in the chain changed the stat, the new value is used.

function (Custom Commands)

Create your own reusable commands. Define a function once, then call it by name anytime. Think of functions as your own custom terminal commands. They can include arguments using $1, $2, etc. so you can reuse the same logic with different values.

Syntax: function <name> <command body>

Arguments: Use $1 for the first argument, $2 for the second, and so on. When you call the function, the values you pass replace the placeholders.

How it works internally: When you call a function, the terminal replaces $1, $2, etc. with the actual values you passed, then runs the resulting commands. The body can contain multiple commands separated by ;. Functions can call other functions.

REPO.crypt (Bank)

The REPO.crypt tab is your secure storage vault. Deposit credits, data, BTC, and bulkheads to keep them safe. But beware — TAXES apply on withdrawals!

Deposits

• Depositing is free — no tax on putting money in.

• Use "DEPOSIT ALL" to quickly store all your current resources.

• Bulkheads can be deposited individually or all at once.

Withdrawals

• All withdrawals are taxed at the current tax rate.

• The tax is deducted from the amount you withdraw.

• Example: If you withdraw 100 credits at 50% tax, you receive 50 credits.

TAX System

The tax rate starts at 13% and increases based on how fast you accumulate wealth:

• Safe (13-30%): Normal gameplay, minimal tax impact.

• Caution (30-60%): Your income speed is raising flags.

• Warning (60-100%): Heavy taxation — slow down!

• Debt Zone (100-150%): Withdrawals cost MORE than you receive. You're losing money!

• Maximum (150-270%): Total economic collapse. Every action drains you.

How tax increases:

• Accumulating large amounts of credits, data, or BTC quickly spikes the tax rate.

• Your total wealth (active + deposited) contributes to the base rate.

• The tax rate slowly decays over time if you play conservatively.

Strategy:

• Deposit frequently to keep active resources low.

• Withdraw slowly over time rather than in large chunks.

• Watch the tax bar in REPO.crypt — if it's red, stop accumulating!

• Tax warnings appear automatically when rates spike.

Registers an event handler that automatically runs a command when a stat meets a condition.

Stats: energy, credits, data, level, xp

Operators: >=, <=, >, <, ==, =, !=

Behavior: Fires once when the condition becomes true. Resets when condition becomes false, allowing it to fire again.

No energy cost.

// Auto-sleep when energy runs out
on(energy <= 0) { run sleep() }

// Auto-compile when data is high
on(data >= 100) { run compile() }

// Auto-hack when credits are low
on(credits <= 10) { run hack() }

Strategy: Use on() to automate repetitive actions. The handler persists across code runs and checks after every mine, sleep, compile, hack, scan, and print action.
